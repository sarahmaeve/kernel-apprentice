$ B4 ./events-and-histograms

ask a question; the kernel counts the answer

B1–B3 traced functions. But the kernel also exposes tracepoints — stable, named, structured events with typed fields — and can aggregate them in-kernel with hist triggers, a tiny query language that turns a million events into a sorted table of counts. No userspace, no overhead.

make check LESSON=B4-events-and-histograms

The shipped trigger.hist keys by ___ — not a real field — so the kernel rejects it and there's no histogram: RED until you key it right.

Needs the Module B kernel. The check detects a stale one and tells you to make kernel.

[01]

./THE EVENTS INTERFACE

Every tracepoint lives under /sys/kernel/tracing/events/<subsystem>/<event>/:

filewhat it is
formatthe event's fields + types — read this to know what you can key/filter on
enableecho 1 to start recording
filtera predicate, e.g. echo 'id == 257' > filter (only openat)
triggerattach actions — including histograms

Subsystems: syscalls / raw_syscalls, sched, irq, net, block, ext4. set_event_pid scopes all events to one task (like set_ftrace_pid did for functions).

[02]

./THE CHALLENGE

raw_syscalls/sys_enter fires on every syscall and carries an id field (the syscall number). In trigger.hist, write a hist trigger that buckets by syscall id. A hist trigger is a one-liner — hist:keys=<field>:sort=<field> — and once it's right the run produces:

# trigger info: hist:keys=id:vals=hitcount:sort=hitcount [active]

{ id:          0 } hitcount:         17     <- read(2)
{ id:        257 } hitcount:         18     <- openat(2)
{ id:         16 } hitcount:         16     <- ioctl(2)
...
Totals:
    Hits: 600
    Entries: 30

Pick a tracepoint, pick a key, read the table. Two keys for a breakdown; :sort=hitcount to rank; a filter to scope.

[03]

./GRADUATED HINTS

Hint 1 — what can I key on?

A hist key must be a field of the event. List them: cat /sys/kernel/tracing/events/raw_syscalls/sys_enter/format — the lines starting field:.

Hint 2 — which field is the syscall number?

It's id (a long). The other field is args (the syscall's arguments).

Hint 3 — the solution 
hist:keys=id:sort=hitcount

Put that line in trigger.hist (replacing the keys=___ placeholder).
