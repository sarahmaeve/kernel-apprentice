what calls what — on a live kernel, no rebuild

Lesson 04 proved the kernel will narrate one function. B1 teaches you to follow that narration through the call graph and back to the caller — the foundation of the whole tracing track. Four moves, all echo-into-tracefs.

make check LESSON=B1-follow-the-path

Everything runs against /sys/kernel/tracing on the live kernel — nothing is compiled in.

Module B note — B1 and B2 run on the stock kernel, but B3 (latency) and B4 (histograms) need extra tracers compiled in. If you built your kernel before Module B landed, run make kernel once to pick them up (incremental).

[01]

./READ — the four moves

A — function_graph: the call graph

 # CPU  DURATION              FUNCTION CALLS
 1)               |  __do_sys_getpid() {
 1)   0.180 us    |    __task_pid_nr_ns();
 1)   0.940 us    |  }

The { … } nesting is the path; DURATION is where time went. getpid is shallow on purpose — graph vfs_* for real depth.

B — set_ftrace_filter, with a wildcard

echo '*getpid*' > set_ftrace_filter   # resolves against available_filter_functions
__x64_sys_getpid
__do_sys_getpid

C — who calls it (func_stack_trace)

   __do_sys_getpid <-__x64_sys_getpid   <- the '<-' column is the immediate caller
 => __do_sys_getpid
 => __x64_sys_getpid
 => do_syscall_64                       <- the whole chain, back to the syscall entry
 => entry_SYSCALL_64_after_hwframe

D — trace vs trace_pipe

cat trace is a re-readable snapshot of the ring buffer; cat trace_pipe consumes as it streams. Knowing which to use is half of not losing your data.

[02]

./TOUCH — re-aim it yourself

In make shell, boot a guest and point the tracer somewhere new:

T=/sys/kernel/tracing
echo function_graph > $T/current_tracer
echo 'vfs_*'        > $T/set_graph_function        # graph the VFS layer
grep -c .           $T/available_filter_functions  # how many functions you *could* trace
echo '*alloc*'      > $T/set_ftrace_filter         # a different wildcard
[03]

$ VERIFY

Green when the captured log shows:

  • the function_graph call path GRAPH
  • a func_stack_trace caller stack CALLER

i.e. you can follow the path down and trace it back up.
