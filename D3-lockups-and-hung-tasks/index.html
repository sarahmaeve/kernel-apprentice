$ D3 ./lockups-and-hung-tasks

the kernel volunteers the evidence

When a CPU stops scheduling or a task stops running, the kernel tells you, unprompted. Two detectors ship enabled on production distro kernels: the soft-lockup watchdog (a CPU hasn't scheduled in N seconds) and khungtaskd (a task stuck in uninterruptible D sleep). A specimen module stalls both ways on purpose; you read both reports.

make check LESSON=D3-lockups-and-hung-tasks

echo spin > /proc/d3-stall hogs a CPU for 6s with preemption off (watchdog fires); echo hang > /proc/d3-stall parks a kthread in D (khungtaskd reports it). Both are reports, not crashes — the box keeps running. Needs the Module D config in the base kernel: the check says make kernel if you're stale.

[01]

./READ — two report shapes

watchdog: BUG: soft lockup - CPU#1 stuck for 6s! [busybox:67]
Modules linked in: d3_stall(O)
RIP: 0010:d3_write.cold+0x3d/0x74 [d3_stall]
Call Trace: ... proc_reg_write ... vfs_write ...

INFO: task d3-hung:70 blocked for more than 3 seconds.
"echo 0 > /proc/sys/kernel/hung_task_timeout_secs" disables this message.
task:d3-hung         state:D stack:15000 pid:70 ...
Call Trace: ... d3_hung_fn+0x42/0x4c [d3_stall] ...
  • soft lockup = a CPU problem — something running won't let go; the report names the CPU, the guilty RIP, and taints with L.
  • hung task = a task problem — something sleeping won't wake; khungtaskd delivers what H2 taught you to pull by hand with sysrq w, unprompted.
  • A hard lockup (interrupts off too) has its own NMI-based detector — same idea, one rung worse; our TCG guest doesn't model it, so it stays prose.
[02]

./TOUCH — the knobs production tunes

echo 2 > /proc/sys/kernel/watchdog_thresh           # soft lockup at ~2x this
echo 3 > /proc/sys/kernel/hung_task_timeout_secs    # khungtaskd's patience
# real fleets also set: kernel.softlockup_panic / hung_task_panic = 1 —
# promote a report to a panic, then kdump gives you a corpse (lesson E1).

These are real fleet-tuning knobs, not lesson props — the check sets them in the guest so both detectors fire within seconds instead of minutes.
