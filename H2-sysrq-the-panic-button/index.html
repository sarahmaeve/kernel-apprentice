$ H2 ./sysrq-the-panic-button

the kernel's emergency intercom

When a box is so wedged that ssh is dead and nothing in userspace answers, one listener remains: the kernel itself. SysRq turns single letters into state dumps on the console, no matter how bad things are. A module parks a kthread in D state on purpose — the "kill -9 does nothing" state — and you pull its location out of the kernel. Then you press the real panic button.

make check LESSON=H2-sysrq-the-panic-button

The check walks t (every task: state + stack), w (only the blocked/D tasks — the first letter you press on a wedged box), m (memory state) and l (backtraces of all active CPUs) via /proc/sysrq-trigger. On real hardware these are keyboard chords or serial-console breaks — which is why they work when nothing else does. The kernel.sysrq mask gates only the keyboard path; a root write to the /proc trigger always works.

 h2_wedge_fn+0x42/0x50 [h2_wedge]
 kthread+0x10a/0x200
 ret_from_fork+0x17e/0x1a0
  • state:D — TASK_UNINTERRUPTIBLE: not runnable, and signals — kill -9 included — stay pending until it next runs, which is exactly what it won't do. Fleeting around I/O normally; a task that stays D is the classic wedged-box symptom.
  • ppid:2 — parent is kthreadd, mother of all kernel threads: not a userspace process at all.
  • the trace, top down — __schedule/schedule is how it sleeps; the first real frame below — h2_wedge_fn — is who chose to. ? frames are stack leftovers (E0); __pfx_* is function padding — skip both.

./TOUCH — the stack file, then the button

$ cat /proc/66/stack          # scheduler frames filtered for you
[<0>] h2_wedge_fn+0x42/0x50 [h2_wedge]
[<0>] kthread+0x10a/0x200
[<0>] ret_from_fork+0x17e/0x1a0

$ echo c > /proc/sysrq-trigger
sysrq: Trigger a crash
Kernel panic - not syncing: sysrq triggered crash
CPU: 1 UID: 0 PID: 1 Comm: init Tainted: G    O    6.18.33 #8 ...

/proc/<pid>/stack (root-only) is the fastest answer to "where is this process stuck?" — top frame = the function that went to sleep. And c panics the kernel on command, in whatever context pressed the button (Comm: init — our init script). Nothing survives; QEMU exits (panic=-1 + -no-reboot). With kdump configured, this is how a hung box becomes a corpse you can autopsy — lesson E1, planned.
